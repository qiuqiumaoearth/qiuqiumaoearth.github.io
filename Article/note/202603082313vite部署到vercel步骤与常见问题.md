# 部署到 Vercel 步骤与常见问题

## 步骤

1. 登录

   打开<https://vercel.com>

   用 GitHub 登录

2. 导入项目

   点击`Add New Project`

   选择你的 GitHub 仓库

3. 自动识别

   Vercel 会自动识别 Vite

   配置一般是：

   ```txt
   Framework: Vite
   Build Command: npm run build
   Output Directory: dist
   ```

4. 点击 Deploy

   30 秒部署完成。

访问：<https://你的项目.vercel.app>

## 常见问题

### 路由问题

Vercel 默认使用 **HTML5 History 模式**，而 Vite 生成的静态文件默认是
**单页应用**，不支持直接刷新。

#### 解决方法

1. 配置 Vite 路由

   在 `vite.config.js` 中添加：

   ```js
   export default defineConfig({
     base: '/' //非常重要
   })
   ```

2. 配置 Vercel 路由

   在 `vercel.json` 中添加：

   ```json
   {
     "rewrites": [
       {
         "source": "/(.*)",
         "destination": "/"
       }
     ]
   }
   ```

### 浏览器安全策略：Mixed Content（混合内容）

```txt
你的网页
<https://hm-shopping-rho.vercel.app>
↓
请求接口
<http://smart-shop.itheima.net>
```

浏览器规则是：

> HTTPS 页面 ❌ 不能请求 HTTP 接口

所以被浏览器直接拦截了。

#### 解决方案:把接口改成 HTTPS

看看这个接口是否支持 HTTPS：

<http://smart-shop.itheima.net/index.php?s=/api/page/detail&pageId=0>

改成：

<https://smart-shop.itheima.net/index.php?s=/api/page/detail&pageId=0>

如果服务器支持 HTTPS，问题会立刻解决。

在你的 axios 配置里改：

`baseURL: 'https://smart-shop.itheima.net'`

### 子目录从当前 Git 仓库中独立

例如现在结构可能是：

```txt
FrontEnd/
├─ 项目A
├─ 项目B
└─ elementProject   ← 你想单独变成一个 Git 仓库
```

而 FrontEnd 这个大目录已经是一个 Git 仓库,想让 `elementProject` 脱离父仓库。

#### 解决方法:脱离父仓库

方案一:

1. 进入这个子项目 `cd elementProject`
2. 删除父仓库的 Git 记录

   删除 .git 目录：`rm -rf .git`

   `Windows（Git Bash）`也一样：`rm -rf .git`

   作用：删除旧 Git 仓库记录,但不会删除代码。

3. 初始化新的 Git 仓库 git init 连接新的远程仓库

   例如你的仓库：<https://github.com/xxx>

   执行：`git remote add origin <https://github.com/xxx>`

4. 提交代码

   ```bash
   git add . git commit -m "init element project"
   git branch -M main
   git push -u origin main
   ```

这样：

```txt
elementProject
      ↓
独立 Git 仓库
```

方法二（保留提交历史，高级用法）

如果你想 保留这个文件夹的 Git 历史，需要用：

```bash
git subtree split
```
