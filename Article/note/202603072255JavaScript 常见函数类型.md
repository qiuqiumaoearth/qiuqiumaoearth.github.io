## 1 函数声明（Function Declaration）

最标准的函数写法。

```js
function foo() {
  console.log('hello')
}
```

特点：

> 会 函数提升（hoisting）优先级高于var

可以在声明前调用

```js
foo()

function foo(){ console.log("hello") } ）
```

## 2 函数表达式（Function Expression）

函数作为一个 值赋给变量。

```js
const foo = function () {
  console.log('hello')
}
```

特点：

不会提升函数体

只有变量提升

例如：

```js
foo()

var foo = function () {
  console.log('hello')
}
```

会报错：

```
foo is not a function
```

## 3 立即执行函数（IIFE）

也叫：

```
Immediately Invoked Function Expression
```

写完就执行。

```js
;(function () {
  console.log('立即执行')
})()
```

或者

```js
;(function () {
  console.log('立即执行')
})()
```

作用：

创建 独立作用域

防止 变量污染

以前在 模块化出现前 很常见。

## 4 箭头函数（Arrow Function）

ES6 新语法。

```js
const foo = () => {
  console.log('hello')
}
```

特点：

1️⃣ 没有自己的 this 2️⃣ 不能当构造函数 3️⃣ 没有 arguments

例如：

```js
const sum = (a, b) => a + b
```

## 5 构造函数（Constructor Function）

用于 创建对象实例。

```js
function Person(name) {
  this.name = name
}

const p = new Person('Tom')
```

特点：

必须配合 new

this 指向实例对象

## 6 Generator 函数（生成器函数）

ES6 新增。

写法：

```js
function\* foo(){ yield 1 yield 2 }
```

执行：

```js
const it = foo()

it.next()
```

返回：

```js
{value:1, done:false}
```

作用：

控制函数 暂停与继续执行

async/await 的底层实现之一

## 为什么函数表达式不能提前调用？

你可以这样回答：

```txt
因为函数表达式本质上是一个变量赋值，例如 varfoo=function(){}。
在 JavaScript 的执行上下文创建阶段，var 只会提升变量声明并赋值为 undefined，函数体不会提升.
因此在赋值之前调用 foo()，相当于调用 undefined()，就会报 TypeError。
```
