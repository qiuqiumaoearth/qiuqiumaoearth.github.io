## 请你区分一下 var、let、const

- var、let、const 的区别主要体现在作用域、变量提升和赋值特性上。
- var 是函数作用域，存在变量提升，声明前访问得到 undefined；let 和 const 是块级作用域，也会提升，但在声明前处于暂时性死区，不能访问。
- 另外，var 允许重复声明，let 和 const 不允许。
- 赋值方面，let 可以重新赋值，const 不可以，但如果 const 保存的是对象，修改对象属性是可以的，因为它没有改变引用地址。

## 说一下什么是暂时性死区？为什么 let 和 const 会有这个机制？

- 暂时性死区是从作用域开始到变量声明之间的区域,再这个区域访问变量会报错.
- 暂时性死区是指从块级作用域开始，到 let 或 const 声明之前的这段区域。
- 在这段区域内，变量虽然已经在作用域中存在，但不能被访问，访问就会报错。
- 这样设计主要是为了避免像 var 那样因为变量提升导致声明前访问得到 undefined，从而提高代码的安全性和可读性。
- 所以可以理解为：let 和 const 也有提升，但在声明前不会初始化。

## let、const 有变量提升吗？如果有，为什么声明前不能访问？

- let 和 const 也有变量提升，但和 var 不一样。
- var 提升后会初始化为 undefined，所以声明前访问不会报错；let 和 const 提升后不会初始化，在声明前处于暂时性死区，所以不能访问。
- 也就是说，它们不是“不提升”，而是“提升了但不能提前用”。

## 说一下闭包是什么？闭包有什么作用和缺点？

- 闭包就是内部函数引用了外部函数作用域中的变量，导致外部函数执行结束后，这些变量仍然可以被访问。
- 它本质上是函数和其词法作用域的组合。
- 闭包常见作用是实现数据私有化、保存状态、做模块化封装。
- 缺点是如果滥用，可能导致变量不能及时被垃圾回收，增加内存开销。

## 闭包为什么会造成内存泄漏？它一定会造成内存泄漏吗？

- 闭包可能导致内存占用增加，是因为内部函数引用了外部变量，只要这个闭包还被引用，这些变量就是可达的，垃圾回收器就不会回收它们。
- 但闭包本身不等于内存泄漏，它只是可能让某些数据存活时间更长。
- 如果闭包是业务上正常需要的，用完后也及时释放引用，那就没有问题。
- 真正的风险是在不再需要时还长期持有闭包，导致内存不能及时释放。

## 说一下 JavaScript 中的原型和原型链

- 在 JavaScript 里，每个构造函数都有一个 prototype 属性，它指向一个原型对象。
- 通过这个构造函数创建出来的实例对象，内部会有一个指针指向这个原型对象，通常可以理解为
  **proto** 指向构造函数的 prototype。
- 原型对象的作用主要是实现属性和方法的共享。
- 比如某个方法定义在原型上，那么这个构造函数创建出来的所有实例都可以访问这个方法，不需要每个实例都复制一份。
- 原型链指的是对象在查找属性或方法时的查找路径。
- 当访问一个对象的属性时，会先在对象自身上找；如果没找到，就沿着它的原型对象继续找；如果还没找到，就继续往上一层原型找，直到 Object.prototype，再往上就是 null。
- 这种一层层向上查找形成的链式结构，就叫原型链。
